# ADR-0004: Модульный монолит на Go + 2 sidecar'а

**Статус:** Accepted
**Дата:** 2026-05-06
**Принимающий:** platform team

## Контекст

Выбор архитектурного стиля для backend: монолит / модульный монолит / микросервисы.

## Альтернативы

- A: Монолит — простота, но размывание границ модулей.
- B: Модульный монолит — единая deployable-единица + sidecar'ы для платформенно-обусловленных компонентов.
- C: Микросервисы — высокая операционная сложность, не оправдана для команды текущего размера.

## Решение

Модульный монолит + telephony-bridge + recording-uploader + cmd/worker.

## Обоснование

См. раздел 5 первоначального обсуждения и раздел 5 этого документа.
